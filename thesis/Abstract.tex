SQL:2016 introduced the \texttt{MATCH\_RECOGNIZE} clause to support row pattern recognition, used to search for a specific pattern in ordered partitions of rows using regular expressions, which allows data analysts to detect patterns according to a user-defined pattern. This capability matters in many domains, such as financial monitoring, climate analysis, security logs, sensor analytics, and e-commerce, where significant information often consists of a sequence of events rather than an isolated record. Several database systems, including Oracle, Snowflake, Trino, and Apache Flink, now provide row pattern recognition, but Pandas lacks native support, so analysts often simulate \texttt{MATCH\_RECOGNIZE} with shifted columns, manual loops, and custom state management, which makes the code longer, less readable, and more error-prone. To close this gap, the proposed engine parses SQL-based \texttt{MATCH\_RECOGNIZE} queries from a string, constructs a structured query representation, validates the supported row-pattern clauses, translates pattern expressions into finite automata, and executes the resulting matcher over ordered DataFrame partitions, covering the evaluated Feature~R010-style scope: \texttt{PARTITION BY}, \texttt{ORDER BY}, \texttt{PATTERN}, \texttt{DEFINE}, \texttt{MEASURES}, skip policies, output modes, quantifiers, alternation, grouping, anchors, exclusions, \texttt{PERMUTE}, and selected navigation and aggregate functions. The system was evaluated through five methods: correctness against Trino and Oracle reference outputs, SQL:2016 feature coverage, usability, and performance and scalability experiments on the Amazon UK Products~2023 catalog, from 100\,K-row prefixes up to the full 2.2-million-row dataset. The engine reproduced the reference outputs row-for-row in all thirty cross-system benchmark cells and recorded the lowest average execution time of the three compared systems (0.20\,s, versus 1.04\,s for Oracle and 1.98\,s for Trino) with approximately linear scaling. These results demonstrate that bringing \texttt{MATCH\_RECOGNIZE}-style functionality to Pandas can bridge SQL and Python workflows, making sequential data analysis more readable, portable, and productive.
