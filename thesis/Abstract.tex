SQL:2016 introduced the \texttt{MATCH\_RECOGNIZE} clause to support row pattern recognition, allowing data analysts to detect ordered sequences of rows that match according to a user-defined pattern. This feature is useful in many domains, such as financial monitoring, climate analysis, security logs, sensor analytics, and e-commerce, where significant information often consists of a sequence of events rather than an isolated record.

Several database systems, including Oracle, Snowflake, Trino, and Apache Flink, now provide row pattern recognition functionality, but Pandas lacks native support for row pattern recognition.

Consequently, analysts often reconstruct sequential patterns with shifted columns, manual loops, and custom state management, which makes the code longer, less readable, and more prone to errors.

This thesis examines SQL:2016 \texttt{MATCH\_RECOGNIZE}-style behavior over Pandas DataFrames. The proposed engine parses SQL-based \texttt{MATCH\_RECOGNIZE} queries from a string, constructs a structured query representation, validates the supported row-pattern clauses, translates pattern expressions into finite automata, and executes the resulting matcher over ordered DataFrame partitions.

The implementation supports the evaluated Feature~R010-style scope, including \texttt{PARTITION BY}, \texttt{ORDER BY}, \texttt{PATTERN}, \texttt{DEFINE}, \texttt{MEASURES}, skip policies, output modes, quantifiers, alternation, grouping, anchors, exclusions, \texttt{PERMUTE}, and selected navigation and aggregate functions.

The system was evaluated through targeted correctness tests, comparisons with Trino and Oracle reference outputs, and performance experiments on samples from the Amazon UK Products~2023 catalog.

The thesis demonstrates that bringing \texttt{MATCH\_RECOGNIZE}-style functionality to Pandas can bridge SQL and Python workflows, making sequential data analysis more readable, portable, and productive.
